Wappalyzer.webpage

package documentation
(source)

Wappalyzer.WebPage uses BeautifulSoup4 by default and fallback to standard library if it's not available.

The following objects are importable form this module: WebPage, IWebPage, ITag.

NoteYou can directly use/subclass one of the WebPage classes provided in modules _bs4 and _stdlib if you'de like more control. Alternatively, your can write your own WebPage from scratch by suclassing the IWebPage interface.
Module _bs4 Implementation of WebPage based on bs4, depends on lxml.
Module _common Containers for a Web page and it's components. Wraps only the information strictly necessary to run the Wappalyzer engine.
Module _stdlib Implementation of WebPage based on the standard library.
